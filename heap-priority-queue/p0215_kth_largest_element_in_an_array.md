# 215. Kth Largest Element in an Array

## 题目

给定整数数组 `nums` 和整数 `k`，返回数组中第 `k` 大的元素。

这里的第 `k` 大是排序后的第 `k` 个位置，不是第 `k` 个不同的元素，因此重复值也要参与排名。

例如：

```text
nums = [3, 2, 1, 5, 6, 4]
k = 2

从大到小：6, 5, 4, 3, 2, 1
第 2 大：5
```

## 识别题型

题目只要求：

```text
第 k 大元素
```

并不要求得到整个有序数组，因此可以考虑用固定大小堆，只维护可能影响答案的 `k` 个元素。

## 核心思路

遍历数组，并维护一个大小不超过 `k` 的最小堆：

```text
堆中始终保留目前见过的最大 k 个元素
```

每次把当前数字加入堆：

```python
heapq.heappush(min_heap, num)
```

如果堆中超过 `k` 个元素，就弹出其中最小的元素：

```python
if len(min_heap) > k:
    heapq.heappop(min_heap)
```

遍历结束后，堆中保存整个数组最大的 `k` 个元素，而堆顶是它们中最小的那个：

```text
最大的 k 个元素中的最小值
=
整个数组的第 k 大元素
```

所以答案是：

```python
min_heap[0]
```

## 为什么第 `k` 大使用最小堆

这一点比较反直觉。

假设需要保留最大的 3 个元素，当前候选是：

```text
10, 8, 6
```

其中最弱的候选是 6。如果又遇到 9，我们需要淘汰 6，留下：

```text
10, 9, 8
```

最小堆可以在 O(1) 时间看到当前最弱候选，并在 O(log k) 时间将它弹出。因此：

```text
保留最大的 k 个 -> 使用大小为 k 的最小堆
```

选择堆的方向时，不要只看题目要求最大还是最小，还要先问：

```text
候选过多时，我需要最快淘汰谁？
```

## 堆中维护的不变量

处理完每个数字后，都应满足：

1. `min_heap` 是合法的最小堆。
2. `len(min_heap) <= k`。
3. 堆中保存目前见过的最大 `k` 个元素；不足 `k` 个时保存全部元素。

遍历完成后，题目保证 `k` 合法，因此堆中正好有 `k` 个元素，`min_heap[0]` 就是答案。

## 执行示例

```text
nums = [3, 2, 1, 5, 6, 4]
k = 2
```

| 当前数字 | 淘汰后保留的最大 2 个 | 堆顶 |
| --- | --- | --- |
| 3 | `3` | 3 |
| 2 | `2, 3` | 2 |
| 1 | `2, 3` | 2 |
| 5 | `3, 5` | 3 |
| 6 | `5, 6` | 5 |
| 4 | `5, 6` | 5 |

表格为了便于观察，把堆中元素写成了有序形式；`heapq` 的内部列表并不保证完全有序。

## 代码

```python
import heapq


class Solution(object):
    def findKthLargest(self, nums, k):
        heap = []

        for num in nums:
            heapq.heappush(heap, num)

            if len(heap) > k:
                heapq.heappop(heap)

        return heap[0]
```

## 与 703 的联系

[703. Kth Largest Element in a Stream](p0703_kth_largest_element_in_a_stream.md) 使用完全相同的核心结构：

```text
大小为 k 的最小堆
```

区别是：

| 题目 | 数据形式 | 查询方式 |
| --- | --- | --- |
| 215 | 数组一次性给出 | 遍历结束后返回一次 |
| 703 | 数据不断加入 | 每次 `add` 后都要返回 |

215 可以看作固定数组版本，703 可以看作把同一个不变量持续维护下去的数据流版本。

## 与完整排序比较

完整排序后可以直接返回：

```python
sorted(nums, reverse=True)[k - 1]
```

其时间复杂度为 O(n log n)。

固定大小堆只维护 `k` 个候选：

- 时间复杂度：O(n log k)
- 空间复杂度：O(k)

当 `k` 明显小于 `n` 时，固定大小堆更符合题目只需要一个排名结果的特点。

## 常见错误

### 使用大小为 `k` 的最大堆

最大堆适合快速取出全局最大值，但本题每次超过 `k` 个候选时，需要淘汰的是最小值，所以最小堆更合适。

### 写成 `len(heap) >= k`

这样会在堆大小刚达到 `k` 时就弹出元素，最终只保留 `k - 1` 个。

正确条件是：

```python
if len(heap) > k:
```

### 使用集合去重

重复值也参与排名，不能用 `set` 去重。

### 认为堆中所有元素都有序

最小堆只保证：

```python
heap[0]
```

是堆中最小值，其他位置不保证有序。

## 心得

1. 第 `k` 大与 703 的本质相同：维护最大的 `k` 个元素。
2. 为了在候选过多时快速淘汰最小者，应使用大小为 `k` 的最小堆。
3. 遍历结束时，堆顶是最大的 `k` 个元素中最小的那个，因此就是第 `k` 大。
4. 题目只需要一个排名结果时，不一定要完整排序，可以只维护会影响答案的候选集合。
5. 选择最小堆还是最大堆时，可以从“超过容量后要淘汰谁”反向思考。
