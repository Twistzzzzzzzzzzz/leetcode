# 271. Encode and Decode Strings

## 题目

设计两个方法：

- `encode(strs)`：把字符串列表编码成一个字符串
- `decode(s)`：把编码后的字符串还原成原来的字符串列表

关键要求是：编码后必须能无歧义地还原。

## 如何识别这类题

当题目要求：

```text
多个字符串 -> 一个字符串 -> 再还原成多个字符串
```

并且原字符串里可能包含任意字符时，就要小心普通分隔符。

比如直接用 `"#".join(strs)` 看起来很自然，但如果原字符串本身就包含 `#`，解码时就会分不清：

```text
["a#b", "c"]
```

所以脑海里第一个想法应该是：

```text
每个字符串都要带上自己的长度。
```

也就是用“长度前缀 + 分隔符 + 原字符串”的格式。

## 思路

把每个字符串编码成：

```text
长度#字符串内容
```

例如：

```text
"hello" -> "5#hello"
""      -> "0#"
```

多个字符串拼在一起：

```text
["lint", "code"] -> "4#lint4#code"
```

这里的 `#` 不是用来分隔每个字符串的。

它只是用来分隔“长度数字”和“字符串内容”。

真正决定要读多少字符的是长度。

## 代码

```python
class Codec:
    def encode(self, strs: list[str]) -> str:
        encoded = []

        for word in strs:
            encoded.append(f"{len(word)}#{word}")

        return "".join(encoded)

    def decode(self, s: str) -> list[str]:
        decoded = []
        i = 0

        while i < len(s):
            j = i

            while s[j] != "#":
                j += 1

            length = int(s[i:j])
            start = j + 1
            decoded.append(s[start : start + length])
            i = start + length

        return decoded
```

## 指针解释

解码时，`i` 指向当前字符串长度数字的起点。

`j` 从 `i` 开始往右走，直到遇到 `#`。

```python
length = int(s[i:j])
```

这里 `s[i:j]` 是左闭右开，所以不包含 `#`。

例如：

```text
s = "12#abcdefghijkl"
     i j
```

当 `j` 走到 `#` 的位置时：

```text
s[i:j] == "12"
```

然后从 `j + 1` 开始，读取 `length` 个字符。

读完后，把 `i` 移动到下一个字符串长度数字的起点。

## 为什么需要两个指针找长度

字符串长度可能不止一位。

比如长度是 `3`：

```text
3#abc
```

长度是 `12`：

```text
12#abcdefghijkl
```

所以不能默认只读一个字符作为长度。

必须用指针一直找到 `#`，才能知道完整的长度数字是多少。

## 复杂度

- 时间复杂度：O(n)
- 空间复杂度：O(n)

其中 `n` 是所有字符串内容加上长度标记后的总字符数。

## 心得

1. 必须用两个指针来找长度位数，因为一个字符串的长度可能是 10 以上，不能只看一位数字。
2. `#` 用来隔开长度和字符串内容。
3. `s[i:j]` 是左闭右开，不包含 `j` 位置的字符。
4. 这题最好画图，指针移动比较绕。
5. 识别这类题时，重点看“要把多个东西打包成一个字符串，并且还要无损还原”。第一反应应该是设计一个不会歧义的格式，常见做法是 `长度 + 分隔符 + 内容`。
