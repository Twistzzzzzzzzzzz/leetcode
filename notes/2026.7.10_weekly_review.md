# 本周刷题速复盘（2026.7.4—2026.7.10）

本周共整理 31 道题：栈 5、滑动窗口 8、二分查找 10、链表 8。

这份笔记不复述题目，而是帮助快速回答四个问题：

1. 看到什么信号？
2. 应该套哪种结构？
3. 代码中必须维持什么不变量？
4. 最容易在哪个边界写错？

## 一、5 分钟题型地图

| 专题 | 看到这些信号 | 第一反应 | 本周锚点题 |
| --- | --- | --- | --- |
| 栈 | 最近一次、嵌套、右侧第一个、连续合并 | 保存未解决状态；当前元素负责弹栈结算 | 394、739、901 |
| 滑动窗口 | 连续子串/子数组、最长/最短、至多/至少、定长 | 右端扩张，左端恢复或榨干某个窗口条件 | 3、76、239 |
| 二分查找 | 有序、答案有范围、可行性随参数单调 | 先写 `check`，再确定找第一个 True 还是最后一个 True | 35、875、1011 |
| 链表 | 头结点可能变化、局部改链、倒数位置、环 | dummy 统一边界；先保存后断链；快慢指针制造距离 | 19、92、287 |

只记四句话：

- 栈：栈里保存的是“还没得到答案”的状态。
- 窗口：先说清窗口合法条件，再决定什么时候收缩、什么时候更新答案。
- 二分：不是“数组有序就二分”，而是搜索空间上的判断必须单调。
- 链表：任何改 `next` 的操作，都先保存仍需访问的节点。

## 二、栈：从 LIFO 到单调压缩

### 共同主线

- 撤销最近动作：普通栈，见 71。
- 进入内层前保存外层上下文：上下文栈，见 394。
- 当前元素解决之前尚未解决的元素：单调栈，见 739。
- 当前元素合并之前的整段信息：压缩单调栈，见 901。
- 排序后只需保留压缩状态：单变量模拟栈顶，见 853。

| 题目 | 识别信号与核心套路 | 关键不变量 | 易错点 | 复杂度 |
| --- | --- | --- | --- | --- |
| [71. 简化路径](../stack/p0071_simplify_path.md) | `..` 撤销最近目录；有效目录入栈 | 栈表示当前已化简的绝对路径 | `...` 是普通目录；根目录的 `..` 忽略 | O(n) / O(n) |
| [394. 字符串解码](../stack/p0394_decode_string.md) | `k[...]` 可嵌套；`[` 时保存 `(外层串, 次数)`，`]` 时恢复组合 | 当前层字符串与外层上下文彼此独立 | 次数可能多位；入栈后要同时重置次数和当前串 | O(n+m) / O(n+m)，m 为输出长度 |
| [739. 每日温度](../stack/p0739_daily_temperatures.md) | 找右侧第一个严格更大值；递减栈存下标 | 栈中下标尚未得到答案，对应温度非递增 | 相等不算更暖，所以弹栈条件是 `< current` | O(n) / O(n) |
| [901. 股票价格跨度](../stack/p0901_online_stock_span.md) | 求向左连续 `<= 当前价` 的长度；栈存 `(price, span)` | 每个栈元素代表一整段已压缩区间 | 相等也要合并，所以弹栈条件是 `<= current` | n 次调用 O(n) / O(n)，单次均摊 O(1) |
| [853. 车队](../stack/p0853_car_fleet.md) | 后车追上前车后永久合并；按位置由近到远看独立到达时间 | 只保留前方车队最慢到达时间 | 到达时间相等也合并；只有 `time > slowest` 才新增车队 | O(n log n) / O(n) |

最值得对比：

- 739 求“严格更大”，相等不能弹。
- 901 求“小于等于当前”的连续跨度，相等必须弹。
- 853 中后车到达时间小于等于前车队时间，就会被吸收。

## 三、滑动窗口：先定义合法，再决定收缩时机

### 四类窗口

1. 定长或有界窗口：219、567。
2. 可变窗口求最长：3、424。
3. 可变窗口求最短：209、76。
4. 特殊窗口：239 用单调队列；658 二分窗口起点。

### 最重要的更新时机

- 最长型：加入右端 → 窗口不合法时收缩 → 恢复合法后更新最大值。
- 最短型：加入右端 → 窗口合法时先更新最小值 → 继续收缩榨干答案。
- 定长型：加入和移除必须对称；长度到达 k 后再检查。

| 题目 | 识别信号与核心套路 | 关键不变量 | 易错点 | 复杂度 |
| --- | --- | --- | --- | --- |
| [3. 无重复字符的最长子串](../sliding-window/p0003_longest_substring_without_repeating_characters.md) | 连续、最长、无重复；重复时左端连续移出 | `seen` 恰好表示当前合法窗口 | 收缩要用 `while`，不是只删一次 | O(n) / O(字符集) |
| [76. 最小覆盖子串](../sliding-window/p0076_minimum_window_substring.md) | 最短窗口覆盖 t 的全部频次；满足后不断缩小 | `matched` 统计已满足的字符种类 | 只在某字符需求跨过 0 时改变 `matched` | O(|s|+|t|) / O(t 的字符种类) |
| [209. 长度最小的子数组](../sliding-window/p0209_minimum_size_subarray_sum.md) | 正数数组、连续和至少 target、求最短 | 正数保证右扩和不减、左缩和不增 | 此套路依赖全为正数；满足时先更新再移除 | O(n) / O(1) |
| [219. 存在重复元素 II](../sliding-window/p0219_contains_duplicate_ii.md) | 相同值的下标距离不超过 k | 检查 i 时，集合只含前面最多 k 个元素 | 检查、加入、淘汰的次序会造成 off-by-one | O(n) / O(min(n,k)) |
| [239. 滑动窗口最大值](../sliding-window/p0239_sliding_window_maximum.md) | 每个长度 k 的窗口都要最大值 | deque 存下标，对应值递减，队首是当前最大值 | 必须存下标才能淘汰；过期条件是 `index < left` | O(n) / O(k) |
| [424. 替换后的最长重复字符](../sliding-window/p0424_longest_repeating_character_replacement.md) | 最多替换 k 次，使窗口字符相同 | 所需替换数 = 窗口长度 − 最高频次 | 当前实现左缩后会重算最高频次；别忘同步维护 | O(n) / O(1)，字符集固定为 26 |
| [567. 字符串的排列](../sliding-window/p0567_permutation_in_string.md) | 排列意味着固定长度且频率完全一致 | 窗口长度恒为 `len(s1)`，字符进出状态对称 | 先把超长窗口缩回 k，再检查频率 | O(|s1|+|s2|) / O(1)，字符集固定为 26 |
| [658. 找到 K 个最接近的元素](../sliding-window/p0658_find_k_closest_elements.md) | 有序数组，答案必为长度 k 的连续段 | 二分窗口起点 `[0,n-k]`，比较两侧候选 | 距离相等优先小值，所以只有严格 `>` 才右移 | O(log(n-k)+k) / O(1)，返回结果 O(k) |

658 虽放在滑动窗口目录中，但实现本质是“二分最优窗口起点”，正好连接窗口与二分两个专题。

## 四、二分查找：精确值、边界、答案

### 下手前先回答

1. 搜索空间是什么：下标、数值答案，还是时间戳？
2. `check(mid)` 随 mid 增大是 `False→True` 还是 `True→False`？
3. 要找任意命中、第一个满足，还是最后一个满足？

### 三类模板

```python
# 精确命中：[left, right]
while left <= right:
    mid = (left + right) // 2
    if value(mid) == target:
        return mid
    if value(mid) < target:
        left = mid + 1
    else:
        right = mid - 1
```

```python
# 第一个 True：False ... False, True ... True
while left < right:
    mid = (left + right) // 2
    if check(mid):
        right = mid
    else:
        left = mid + 1
return left
```

```python
# 最后一个 True：True ... True, False ... False
while left < right:
    mid = (left + right + 1) // 2
    if check(mid):
        left = mid
    else:
        right = mid - 1
return left
```

| 题目 | 属于哪类、如何判断 | 易错点 | 复杂度 |
| --- | --- | --- | --- |
| [374. 猜数字](../binary-search/p0374_guess_number_higher_or_lower.md) | 闭区间精确查找，根据 API 排除 mid | `guess(mid)==1` 表示猜小了，应右移 left | O(log n) / O(1) |
| [35. 搜索插入位置](../binary-search/p0035_search_insert_position.md) | lower_bound：找第一个 `>= target` | 右边界是 `len(nums)`，答案可能在末尾 | O(log n) / O(1) |
| [69. x 的平方根](../binary-search/p0069_sqrtx.md) | 找最大的 k，使 `k² <= x`，即最后一个 True | `left=mid` 时必须取右中位数，否则死循环 | O(log x) / O(1) |
| [74. 搜索二维矩阵](../binary-search/p0074_search_a_2d_matrix.md) | 矩阵全局有序，可用虚拟一维下标精确查找 | 只有全局有序才能展平；坐标为 `mid//cols, mid%cols` | O(log(mn)) / O(1) |
| [153. 旋转数组最小值](../binary-search/p0153_find_minimum_in_rotated_sorted_array.md) | 与 right 比较定位旋转点 | `nums[mid] <= nums[right]` 时要 `right=mid`，不能丢掉 mid | O(log n) / O(1) |
| [33. 搜索旋转排序数组](../binary-search/p0033_search_in_rotated_sorted_array.md) | 每次至少一半有序；先判有序半边，再判 target 值域 | 左半有序判断用 `nums[left] <= nums[mid]` | O(log n) / O(1) |
| [81. 搜索旋转排序数组 II](../binary-search/p0081_search_in_rotated_sorted_array_ii.md) | 33 的重复元素版；三端相等时向内缩 | 先检查 mid；大量重复会退化到 O(n) | 平均 O(log n)，最坏 O(n) / O(1) |
| [875. 爱吃香蕉的珂珂](../binary-search/p0875_koko_eating_bananas.md) | 速度越大越可行，找第一个 True | 单堆时间要向上取整：`(pile+speed-1)//speed` | O(n log M) / O(1) |
| [981. 基于时间的键值存储](../binary-search/p0981_time_based_key_value_store.md) | 每个 key 内找最后一个 `timestamp <= query` | 先找第一个 `> query` 再减一；结果为 -1 时返回空串 | set O(1)，get O(log m)，空间 O(N) |
| [1011. 在 D 天内送达包裹](../binary-search/p1011_capacity_to_ship_packages_within_d_days.md) | 容量越大越可行，找第一个 True | 下界是单件最大重量；模拟时不可改变包裹顺序 | O(n log S) / O(1) |

迁移关系：

- 35 是“第一个满足”的基础。
- 69 是“最后一个满足”的基础。
- 981 是“第一个大于查询值，再退一步”。
- 875 与 1011 的二分完全相同，差别只在 `check`：一个统计小时，一个模拟天数。
- 33 与 81 的难点不是二分模板，而是判断哪一半仍然有序。

## 五、链表：dummy、反转、快慢指针、映射

### 四组核心模型

- dummy：统一“头结点可能被删除、替换或重新连接”的边界，见 2、19、21、92。
- 三指针反转：`prev` 是已反转部分，`curr` 是待处理节点，`nxt` 保存未处理部分，见 206、92。
- 快慢指针：固定距离解决倒数位置，速度差解决环，见 19、141、287。
- 旧节点到新节点的映射：把深拷贝视为复制一张指针图，见 138。

| 题目 | 识别信号与核心套路 | 关键不变量 / 易错点 | 复杂度 |
| --- | --- | --- | --- |
| [2. 两数相加](../linked-list/p0002_add_two_numbers.md) | 按位模拟加法；dummy + curr 构造结果，carry 传递状态 | 循环条件要包含 `carry`，否则漏掉最高位进位 | O(max(m,n)) / O(max(m,n))，空间为结果链表 |
| [19. 删除倒数第 N 个结点](../linked-list/p0019_remove_nth_node_from_end_of_list.md) | fast 先从 dummy 走 n 步，再与 slow 同行 | 循环到 `fast.next` 为空，使 slow 停在待删节点前驱 | O(n) / O(1) |
| [21. 合并两个有序链表](../linked-list/p0021_merge_two_sorted_lists.md) | dummy + curr，每次接较小节点，最后接剩余链 | 复用原节点；每次接入后要移动来源指针和 curr | O(m+n) / O(1) |
| [206. 反转链表](../linked-list/p0206_reverse_linked_list.md) | 标准 `prev-curr-nxt` 三指针 | 改 `curr.next` 前必须先保存 `nxt` | O(n) / O(1) |
| [92. 反转链表 II](../linked-list/p0092_reverse_linked_list_ii.md) | 找到 left 前驱，截取并反转区间，再接回两端 | 原 left 节点反转后是区间尾；要保留 right 后继 | O(n) / O(1) |
| [141. 环形链表](../linked-list/p0141_linked_list_cycle.md) | slow 一步、fast 两步，有环必相遇 | 循环条件必须检查 `fast and fast.next`；比较节点身份 | O(n) / O(1) |
| [287. 寻找重复数](../linked-list/p0287_find_the_duplicate_number.md) | 把 `i -> nums[i]` 看成函数图；重复值就是环入口 | 第一阶段找相遇点；第二阶段从 0 与相遇点同速走；返回入口下标/值 | O(n) / O(1) |
| [138. 随机链表的复制](../linked-list/p0138_copy_list_with_random_pointer.md) | 两遍哈希：先创建全部新节点，再复制 next/random | 初始化 `{None: None}`；不能让新节点指向任何旧节点 | O(n) / O(n) |

建议先练 206，再做 92；先理解 141，再做 287。后者都是前者模板的迁移，而不是两道全新的题。

## 六、最容易混淆的 10 个问题

1. 为什么 739 弹 `< current`，901 却弹 `<= current`？
2. 最长窗口为何在恢复合法后更新，最短窗口为何在合法时边缩边更新？
3. 209 为什么必须要求正数？有负数时哪条单调性消失？
4. 239 为什么 deque 必须存下标，而不能只存值？
5. 658 为什么比较的是 `arr[mid]` 与 `arr[mid+k]`，而不是窗口的两个端点？
6. 69 中为什么要用右中位数？
7. 81 为什么可能从 O(log n) 退化为 O(n)？
8. 981 为什么最终答案是 `left - 1`？
9. 92 反转完成后，原 left 节点与原 right 节点分别变成什么位置？
10. 287 中为什么第二阶段要让一个指针回到下标 0？

如果这 10 个问题能不看代码说清楚，本周内容就不只是“做过”，而是已经形成题型模型。
