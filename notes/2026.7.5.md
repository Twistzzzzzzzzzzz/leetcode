# 2026.7.5 刷题总结

## 今日整理的题型

今天主要进入滑动窗口专题，同时补了一道栈题。

## 栈：保存嵌套状态

代表题：

- 394. Decode String

核心思路：

遇到嵌套结构时，可以用栈保存进入下一层之前的状态。

例如 `3[a2[c]]` 中，遇到 `[` 时保存：

```python
(之前的字符串, 重复次数)
```

遇到 `]` 时弹出状态并合并。

关键点：

- 多位数字可以用 `num = num * 10 + int(ch)` 累积。
- 栈里保存的是“回到上一层时需要恢复的信息”。
- 不要一开始就写复杂的双指针切片，先想状态如何入栈、出栈。

## 滑动窗口：基础框架

滑动窗口本质是维护一个连续区间：

```python
nums[left:right + 1]
s[left:right + 1]
```

通用最长型模板：

```python
for right in range(n):
    加入右端元素

    while 窗口不合法:
        移除左端元素
        left += 1

    更新最大长度
```

通用最短型模板：

```python
for right in range(n):
    加入右端元素

    while 窗口已经满足条件:
        更新最小长度
        移除左端元素
        left += 1
```

核心区别：

- 最长型：先收缩到合法，再更新最大值。
- 最短型：一旦满足条件，先更新答案，再尝试缩小。

## 固定长度窗口 + Set

代表题：

- 219. Contains Duplicate II

适用场景：

判断最近 `k` 个元素里是否出现过某个值。

核心思路：

维护一个大小最多为 `k` 的集合窗口。

关键点：

- 当前元素加入后，窗口是给下一个下标使用的。
- 所以收缩可以放在加入当前元素之后。
- `enumerate(nums)` 可以同时拿到下标和值。

## 可变长度窗口：无重复

代表题：

- 3. Longest Substring Without Repeating Characters

适用场景：

题目出现：

```text
最长子串 + 不重复 + 连续
```

核心状态：

```python
seen = 当前窗口里的字符
```

不合法条件：

```python
s[right] in seen
```

关键点：

- 如果当前字符重复，就不断移除 `s[left]`，直到窗口重新合法。
- 要用 `while`，因为一次重复可能需要移除多个左侧字符。
- 窗口长度可以直接写 `right - left + 1`。

## 可变长度窗口：替换次数限制

代表题：

- 424. Longest Repeating Character Replacement

核心公式：

```python
窗口长度 - 窗口内最高频字符次数 <= k
```

理解方式：

窗口里保留出现最多的字符，其他字符都替换成它。

所以需要替换的数量是：

```python
right - left + 1 - max_freq
```

关键点：

- 字符频率用字典维护。
- `max(freq.values())` 可以拿到窗口内最高频次数。
- `max(freq, key=freq.get)` 可以拿到最大 value 对应的 key。
- 不要把变量命名为 `dict`，因为 `dict` 是 Python 内置类型名。

## 固定长度窗口 + 频率抵消

代表题：

- 567. Permutation in String

适用场景：

判断一个字符串是否包含另一个字符串的排列。

核心思路：

排列长度不变，所以窗口长度固定为：

```python
k = len(s1)
```

用字典记录 `s1` 的字符需求：

- 右端进入窗口：需求减一
- 左端离开窗口：需求加一
- 窗口长度等于 `k` 时，如果所有需求都是 `0`，说明找到排列

容易混的点：

- `while right - left + 1 >= k` 和 `while right - left + 1 > k` 不能只改符号。
- `>= k` 常见流程是先检查长度为 `k` 的窗口，再收缩。
- `> k` 常见流程是先收缩到不超过 `k`，再检查长度是否等于 `k`。

## 可变长度窗口：最短子数组

代表题：

- 209. Minimum Size Subarray Sum

适用场景：

正数数组里找：

```text
和至少为 target 的最短连续子数组
```

核心状态：

```python
window_sum
```

关键点：

当 `window_sum >= target` 时：

```python
answer = min(answer, right - left + 1)
window_sum -= nums[left]
left += 1
```

答案更新必须放在缩小窗口之前。

因为缩小前的窗口已经满足条件，是一个合法答案。

问最小值时，可以用：

```python
answer = float("inf")
```

最后如果没有更新过，再返回题目要求的默认值。

## 二分固定窗口左边界

代表题：

- 658. Find K Closest Elements

这题很特别，是：

```text
二分查找 + 固定长度窗口
```

最终答案一定是长度为 `k` 的连续窗口：

```python
arr[left:left + k]
```

所以真正二分的不是最接近 `x` 的元素下标，而是：

```text
最优窗口左边界
```

对于候选窗口：

```python
arr[mid:mid + k]
```

它的左边界是：

```python
arr[mid]
```

它右边外面第一个候选元素是：

```python
arr[mid + k]
```

比较：

```python
x - arr[mid]
arr[mid + k] - x
```

如果：

```python
x - arr[mid] > arr[mid + k] - x
```

说明左边界 `arr[mid]` 比右边候选 `arr[mid + k]` 离 `x` 更远，窗口太靠左，应该右移：

```python
left = mid + 1
```

否则窗口不该右移：

```python
right = mid
```

关键点：

- `mid` 是候选窗口左边界，不是最接近 `x` 的元素下标。
- 不要用窗口 `sum` 判断接近程度。
- 总和接近 `k * x`，不代表窗口里的每个元素都更接近 `x`。

## 今日最重要的收获

1. 滑动窗口的第一步是定义窗口状态。
2. 最长型和最短型的答案更新位置不同。
3. 固定窗口里，检查答案和收缩窗口的顺序必须匹配。
4. 字符串排列、Anagram 这类问题，本质上是固定长度窗口加频率统计。
5. 有序数组里的固定长度最优窗口，可以考虑二分窗口左边界。
6. 看到 `mid` 不一定就是元素下标，也可能是窗口起点、答案边界或搜索空间里的候选答案。
