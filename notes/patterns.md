# 刷题模式笔记

## 数组和哈希表

- 优化 = 找出暴力解中重复做的事情，然后用合适的数据结构或状态把它保存下来。
- 如果题目在问“数组中是否存在两个数或多个数满足某种关系”，可以考虑用哈希表保存已经见过的信息，或者保存后面需要匹配的关系。
- 写哈希表前先明确字典含义：是存 `value -> index`，还是 `complement -> index`。
- 分类题的一大思路是为每一类找到一个统一的类别标识。例如对于 Anagram，这个标识可以是排序后的字符串。
- 遇到“多个类别、每个类别有多个元素”的分组题，可以用 HashMap 当桶：`key` 表示类别，`value` 保存这个类别里的元素列表。
- 分组题不要优先两两比较，先想每个元素应该被放进哪个桶。
- 手写 HashSet / HashMap 时，先明确三件事：哈希函数怎么把 `key` 映射到下标，桶里存什么，冲突时怎么处理。
- HashSet 的桶里只存 `key`；HashMap 的桶里要存 `[key, value]`，并且 `put` 遇到旧 `key` 时要更新旧值。
- 题目出现“绝对多数”时，可以考虑投票和抵消思想：不同元素互相抵消，最后留下来的仍然是多数元素。
- 如果题目要求同时满足多种“不重复”约束，可以一次遍历，同时维护多个集合。
- 二维坐标需要映射成一维类别编号时，可以考虑类似 `(row // size) * group_count + (col // size)` 的公式。
- 遇到 `unsorted array + O(n) time`，并且需要频繁判断某个值是否存在，可以优先想到 `set`。
- 连续序列题不要从每个数字都开始扩展，先判断它是不是起点，例如 `num - 1 not in set`。
- 股票题先看交易次数限制：只能一次交易时，维护历史最低买入价；可以多次交易时，收集所有相邻上涨差价。
- 只能交易一次的最大利润题，可以把每一天当成卖出日，用它减去之前见过的最低价格。
- 可以多次交易时，一整段上涨可以拆成很多个相邻上涨小段，利润总和不变。
- 如果答案只可能落在 `1..n+1`，可以考虑把原数组当哈希表：数字 `x` 尽量放到下标 `x - 1`。
- 原地哈希也可以用负号标记：下标表示数字，正负号表示这个数字是否出现过。
- 原地哈希题里的 `while` 往往仍然是 O(n)，因为每次成功交换至少会把一个元素放到正确位置。
- 先确认是否需要保留原始下标。
- 遇到查找「另一个数」时优先考虑哈希表。
- 注意重复元素和空输入。

## 双指针

- 排序后查找 pair 或区间时常用。
- 已排序数组里找两个数满足某个和时，可以用左右双指针：和太小移动左指针，和太大移动右指针。
- 已排序是左右双指针的重要触发点：因为排序让指针移动后的变化方向可预测，例如左指针右移会变大，右指针左移会变小。
- 如果题目要求找若干数之和，并且数组可以排序，且不要求返回原始下标，可以考虑排序加双指针。
- `3Sum` 这类问题的本质是降维：先固定一个数，把剩下部分转成已排序数组里的 `2Sum` 双指针问题。
- `4Sum` 可以继续降维：固定两个数，把剩下部分转成已排序数组里的 `2Sum` 双指针问题。
- 多层固定值时，每一层都要在同一层循环内跳过重复值，例如第二层常见边界是 `second > first + 1`。
- 排序不只是为了双指针，也是为了把相同数字放在一起，方便提前跳过重复值。
- 需要不重复答案时，去重要在生成重复答案之前完成，不要依赖 `if answer not in result` 这种事后补救。
- 排序后做最轻和最重配对时，可以每轮优先处理最重的人；如果最轻都不能和最重配对，最重通常只能单独处理。
- 容器盛水类问题里，面积由宽度和较短边共同决定；宽度变小时，只有移动较短边才可能找到更大的面积。
- 接雨水类问题中，某个位置能接的水由 `min(左边最高, 右边最高)` 决定；可以先用左右最高数组理解，再用双指针优化空间。
- 接雨水双指针的核心是：哪一侧当前最高墙更矮，就先处理哪一侧，因为这一侧的水位已经由较矮墙决定。
- 一左一右适合收缩搜索空间。
- 快慢指针适合原地覆盖、去重、链表环。
- 遇到“反转”“回文”“首尾交换”这类需要两端成对处理的问题，可以先想到一左一右两个指针向中间收缩。
- 左右指针向中间收缩时，循环条件通常写 `while left < right`，因为偶数长度时两个指针会错过，不一定会相等。
- 回文题如果需要跳过无效字符，内层移动指针的 `while` 也要带上 `left < right`，因为移动过程中两个指针可能相遇或交叉。
- 回文题如果允许删除一个字符，第一次不匹配时分别检查“删左边”和“删右边”两个剩余区间。
- 两个输入序列需要交替合并或同步推进时，可以给每个序列一个指针，分别记录当前处理到哪里。
- 合并两个有序数组且第一个数组尾部有空位时，优先从后往前写，避免覆盖第一个数组中还没比较过的有效元素。
- 从后往前合并时，如果最后剩下的是第二个数组，需要继续复制；如果剩下的是第一个数组，通常已经在正确位置。
- 原地删除类题目可以理解为“重建有效前缀”：把要保留的元素依次写到数组最前面，后面的内容不用管。
- 有序数组原地去重可以用快慢指针：`fast` 扫描新元素，`slow` 维护有效区间最后一个位置，最终长度通常是 `slow + 1`。
- 三指针分区题要明确每个区间的含义；如果和右侧未知元素交换，当前指针通常不能立刻前进，需要重新检查换回来的元素。

## 字符串

- 遇到“多个字符串编码成一个字符串，再无损解码回来”的题，普通分隔符可能会和原字符串内容冲突，优先考虑 `长度 + 分隔符 + 内容` 的自描述格式。
- 解析 `长度#内容` 这类格式时，长度可能有多位，先用指针找到分隔符，再用切片读取指定长度的内容。
- 循环构造字符串时，刷题里优先用 `list.append(...)` 收集片段，最后用 `"".join(parts)` 一次性拼成答案。
- `separator.join(iterable)` 里的调用者是分隔符，不是被追加内容的目标字符串；`join` 返回新字符串，不会原地修改原字符串。
- 少量字符串拼接可以用 `+`；简单变量嵌入优先用 f-string；大量循环拼接不优先用反复 `+=`。
- 字符串拼接的详细整理见 [Python 字符串拼接](python_string_concatenation.md)。

## 栈

- 栈的核心是后进先出：最后放进去的元素最先被取出来。
- Python 里刷题通常用 `list` 当栈：`append` 入栈，`pop` 出栈，`stack[-1]` 查看栈顶。
- 看到“最近的”“上一个”“撤销”“删除前一个”这类描述，可以优先考虑栈。
- 括号匹配适合用栈，因为最近打开的括号必须最先关闭。
- 括号匹配可以用字典保存 `右括号 -> 左括号`，减少多分支判断。
- 栈模拟碰撞时，先明确真正会发生碰撞的方向条件；例如小行星碰撞只有 `栈顶向右 && 当前向左` 才会撞。
- 当前元素可能连续影响多个栈顶元素时，通常需要用 `while stack and ...` 反复处理。
- 逆波兰表达式适合用栈：数字入栈，遇到运算符时弹出最近两个数字计算，再把结果放回栈。
- 栈里弹出两个操作数时要注意顺序：先弹出的是右操作数，后弹出的是左操作数。
- 单调栈适合处理“下一个更大元素”“下一个更小元素”这类问题，后续再重点练。
- 单调栈里经常存下标而不是值，因为很多题需要计算距离，例如 `当前下标 - 栈顶下标`。
- 单调栈也可以存元组，例如 `(price, span)`，把已经合并好的连续信息压缩到栈里。
- 单调栈保存的是还没找到答案的元素；新元素来了以后，尝试用 `while stack and ...` 给栈顶元素结算答案。
- 如果当前元素可以覆盖或合并前面一段连续元素，可以考虑用单调栈保存“值 + 已合并长度”。
- 两个列表按下标一一对应时，可以用 `zip(a, b)` 打包成元组列表，再一起排序。
- 车队题可以按位置从靠近终点到远离终点排序，再维护前方车队到达时间；后车时间小于等于前方车队时间时会合并。
- 如果单调栈过程只需要栈顶信息，可以把栈压缩成一个变量；思想仍然是单调栈，实现可以更简单。
- 遇到“追上”“合并”“被前面挡住”这类描述，可以考虑单调栈或单调栈的压缩写法。
- 能合并到已有状态时，不更新主状态；形成新状态时，才更新主状态。
- 栈的详细整理见 [栈基础](stack_basics.md)。

## 排序

- 归并排序的递归结构是：拆成左右两半，分别递归排序，再合并两个有序数组。
- 快速排序的直观写法是：选 pivot，把元素分成小于、等于、大于三组，再递归排序左右两边。
- 额外数组版快速排序更容易理解；原地指针版更省空间，但边界更容易写错。
- 如果题目给了明确且不大的数值范围，可以考虑计数排序：下标表示数字，值表示出现次数。

## 前缀和

- 看到多次查询区间和、矩形区域和，并且查询要求 O(1)，可以想到前缀和：把重复计算提前放到构造阶段。
- 二维前缀和可以理解为一种状态复用：当前格子复用上方、左方、左上方已经算好的结果。
- 二维矩阵题可以在外圈加一圈 `0` padding，统一公式，减少第一行和第一列的边界判断。
- 前缀积和前缀和是同一类思想：提前保存左边累计信息，再结合右边累计信息得到当前答案。
- 遇到“除了自己以外的所有元素”这类题，可以先想能不能拆成“左边贡献”和“右边贡献”。
- 遇到“连续子数组和等于 k”，尤其数组里可能有负数时，可以考虑“前缀和 + 哈希表计数”。
- 前缀和哈希表常见初始化是 `{0: 1}`，用来表示空前缀，处理从下标 0 开始的子数组。

## 滑动窗口

- 适合连续子数组或子串。
- 先定义窗口内需要维护的状态。
- 扩右边界，必要时收左边界。

## 二分查找

- 先明确搜索区间是闭区间还是左闭右开。
- 写循环前确认答案是否可能在边界。
- 判断条件尽量围绕「答案在左边还是右边」来写。
