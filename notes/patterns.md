# 刷题模式笔记

这份文件用于快速复盘“看到题目时该往哪个方向想”。

每道题的详细解释仍然放在对应题目的 `.md` 文件里；这里保留跨题型的识别信号、常用模板和易错点。

## 快速判断表

| 题目信号 | 优先想到 |
| --- | --- |
| 是否出现过、是否重复、快速查找某个值 | `set` / HashMap |
| 两数或多个数满足某种关系 | HashMap 保存已见信息或所需关系 |
| 多个类别，每类多个元素 | HashMap 分组 / 桶思想 |
| 已排序数组里找 pair / 区间 | 左右双指针 |
| 原地删除、原地去重、重建前缀 | 快慢指针 |
| 回文、反转、首尾交换 | 左右指针向中间收缩 |
| 连续子数组 / 子串 + 最长 / 最短 / 满足条件 | 滑动窗口 |
| 子数组和等于 `k`，数组可能有负数 | 前缀和 + HashMap |
| 最近的、上一个、撤销、括号匹配、嵌套结构 | 栈 |
| 下一个更大 / 更小元素 | 单调栈 |
| 固定窗口最大值 | 单调队列 |
| 排序数组中要求 `O(log n)` | 二分查找 |
| 最小速度 / 最小容量 / 最小天数 | 二分答案 |
| 多次查询区间和，要求 `O(1)` | 前缀和 |
| 区间只在起点增加、终点减少，最后按坐标扫描 | 差分数组 / 扫描线 |
| 答案只可能在 `1..n+1` | 原地哈希 |
| 链表删除、插入、合并、反转 | dummy / prev / curr / next |
| 链表找中点、倒数第 n 个、判断环 | 快慢指针 |
| 数组值可以作为下一个下标，且要求 O(1) 空间 | 数组模拟链表 / Floyd 判环 |
| 二叉树深度、高度、路径、左右子树汇总 | DFS（递归或显式栈） |
| 二叉树按层处理、找最近层 | BFS + queue |
| 二叉搜索树查找、第 k 小、验证有序性 | BST 性质 / 中序遍历 / 上下界 |
| 多次插入字符串并查询完整单词或前缀 | Trie / 前缀树 |
| Trie 搜索中存在通配符或多个候选字符 | Trie + DFS 分支搜索 |
| 反复取当前最小值 / 最大值、第 k 大 / Top K | Heap / Priority Queue |
| 合并多个有序序列 | Heap 中为每个来源保留一个候选 |
| 所有子集、组合、排列、分割或棋盘方案 | Backtracking / 回溯 |
| 节点之间存在连接、依赖或状态转移 | Graph + adjacency list |
| 无权图最短步数、传播时间、同时扩散 | BFS / 多源 BFS |
| 课程或任务先后依赖 | Topological Sort |
| 动态合并集合、判断是否已经连通 | Union-Find |

## 通用原则

- 优化 = 找出暴力解中重复做的事情，然后用合适的数据结构或状态把它保存下来。
- 写 HashMap 前先明确字典含义：是 `value -> index`，还是 `complement -> index`，还是 `key -> group`。
- 先确认题目是否要求保留原始下标；如果不要求，排序经常能降低难度。
- 先确认输入是否可能为空，是否有重复元素，是否有负数。
- 不要只记代码模板，要能说清楚状态含义：集合里存什么、字典值代表什么、窗口是否合法、`mid` 代表什么。

## 数组和哈希表

### 识别信号

- 判断某个元素是否出现过、是否重复。
- 在数组中找两个数或多个数是否满足某种关系。
- 题目要求 `O(n)`，并且需要频繁判断某个值是否存在。
- 题目是分组、分类、计数。

### 常用模式

- `set` 适合判断是否出现过，例如 Contains Duplicate。
- HashMap 适合保存关系，例如 Two Sum 里保存 `complement -> index` 或 `value -> index`。
- 分组题用 HashMap 当桶：`key` 表示类别，`value` 保存这一类元素列表。
- Anagram 的统一类别标识可以是排序后的字符串。
- 多种“不重复”约束可以一次遍历，同时维护多个集合，例如行、列、宫。

### 特殊技巧

- 连续序列题不要从每个数字都开始扩展，先判断它是不是起点，例如 `num - 1 not in set`。
- 二维坐标映射成一维编号时，可以用 `(row // size) * group_count + (col // size)`。
- 绝对多数题可以考虑投票和抵消思想：不同元素互相抵消，最后留下的仍然是多数元素。
- 原地哈希：如果答案只可能在 `1..n+1`，可以尝试把数字 `x` 放到下标 `x - 1`。
- 原地哈希也可以用负号标记：下标表示数字，正负号表示这个数字是否出现过。

### 易错点

- 不要事后用 `if answer not in result` 去重，能提前跳过重复值就提前跳过。
- 原地哈希里的 `while` 往往仍然是 O(n)，因为每次成功交换至少放对一个元素。
- HashSet 的桶里只存 `key`；HashMap 的桶里要存 `[key, value]`，并且旧 `key` 要更新旧值。

## 双指针

### 识别信号

- 已排序数组里找两个数满足某个和。
- 题目可以排序，并且不要求返回原始下标。
- 需要从两端成对处理，例如反转、回文、首尾交换。
- 原地删除、原地覆盖、原地去重。

### 左右指针

- 已排序数组找两数之和：和太小移动左指针，和太大移动右指针。
- 已排序是左右双指针的重要触发点，因为移动方向带来的变化可预测。
- 一左一右适合收缩搜索空间。
- 左右指针向中间收缩时，循环条件通常写 `while left < right`。

### 多数之和

- `3Sum` 的本质是降维：固定一个数，把剩余部分变成有序数组中的 `2Sum`。
- `4Sum` 继续降维：固定两个数，再做 `2Sum`。
- 如果题目要求找若干数之和，并且数组可以排序，且不要求返回原始下标，可以考虑排序 + 双指针。
- 排序不仅是为了双指针，也是为了把相同数字放在一起，方便提前去重。

### 快慢指针

- 原地删除类题目可以理解为“重建有效前缀”：把要保留的元素依次写到数组前面。
- 有序数组原地去重：`fast` 扫描新元素，`slow` 维护有效区间最后一个位置，最终长度通常是 `slow + 1`。

### 经典题型

- 合并两个有序数组且第一个数组尾部有空位时，优先从后往前写，避免覆盖还没比较过的有效元素。
- 接雨水中，某个位置能接的水由 `min(左边最高, 右边最高)` 决定。
- 接雨水双指针的核心：哪一侧当前最高墙更矮，就先处理哪一侧。
- 容器盛水中，面积由宽度和较短边决定；宽度变小时，只有移动较短边才可能找到更大面积。

## 字符串

### 编码与解码

- 多个字符串编码成一个字符串再无损解码时，普通分隔符可能和原内容冲突。
- 优先考虑 `长度 + 分隔符 + 内容` 的自描述格式。
- 解析 `长度#内容` 时，长度可能有多位，先用指针找到分隔符，再按长度切片。

### 自定义字典序

- 自定义字符顺序先转换成 `字符 -> 排名` 的 HashMap，再进行 O(1) 排名查询。
- 验证字符串列表是否有序时，只需要逐对检查相邻字符串。
- 两个字符串的顺序由第一个不同字符决定，找到后立即停止比较后续字符。
- 如果较短长度范围内完全相同，则较短字符串必须排在较长字符串前面。
- “已知顺序并验证”是字符串比较；“根据单词关系推导顺序”才需要 Graph + Topological Sort。

### 字符串拼接

- 循环构造字符串时，刷题里优先用 `list.append(...)` 收集片段，最后用 `"".join(parts)`。
- `separator.join(iterable)` 中调用者是分隔符，不是被追加内容的目标字符串。
- `join` 返回新字符串，不会原地修改原字符串。
- 少量拼接可以用 `+`；变量嵌入优先用 f-string；大量循环不优先反复 `+=`。
- 详细整理见 [Python 字符串拼接](python_string_concatenation.md)。

## 栈

### 识别信号

- 最近的、上一个、撤销、删除前一个。
- 括号匹配、嵌套结构、路径回退。
- 表达式计算。
- 下一个更大 / 更小元素。

### 普通栈

- 栈的核心是后进先出。
- Python 里通常用 `list` 当栈：`append` 入栈，`pop` 出栈，`stack[-1]` 看栈顶。
- 括号匹配适合用栈，因为最近打开的括号必须最先关闭。
- 括号匹配可以用字典保存 `右括号 -> 左括号`。
- 逆波兰表达式：数字入栈，遇到运算符时弹出最近两个数字计算，再放回栈。
- 弹出两个操作数时注意顺序：先弹出的是右操作数，后弹出的是左操作数。

### 模拟与嵌套

- 小行星碰撞只在 `栈顶向右 && 当前向左` 时发生，不是异号就碰撞。
- 当前元素可能连续影响多个栈顶元素时，通常用 `while stack and ...`。
- 路径简化中，`..` 可以理解为撤销最近进入的目录。
- 嵌套字符串解码：遇到 `[` 保存 `(之前的字符串, 重复次数)`，遇到 `]` 弹出并合并。
- 多位数字可以逐字符累积：`number = number * 10 + int(char)`。

### 单调栈

- 单调栈适合处理“下一个更大元素”“下一个更小元素”。
- 单调栈保存的是还没找到答案的元素。
- 栈里经常存下标而不是值，因为很多题需要计算距离。
- 也可以存元组，例如 `(price, span)`，把已合并的连续信息压缩到栈里。
- 如果过程只需要栈顶信息，可以把栈压缩成一个变量；思想仍然是单调栈。
- 遇到“追上”“合并”“被前面挡住”，可以考虑单调栈或单调栈压缩写法。
- 详细整理见 [栈基础](stack_basics.md)。

## 排序

- 归并排序：拆成左右两半，分别递归排序，再合并两个有序数组。
- 快速排序：选 pivot，把元素分成小于、等于、大于三组，再递归排序左右两边。
- 额外数组版快速排序更容易理解；原地指针版更省空间，但边界更容易错。
- 题目给了明确且不大的数值范围时，可以考虑计数排序：下标表示数字，值表示出现次数。

## 前缀和

### 识别信号

- 多次查询区间和、矩形区域和，并且查询要求 O(1)。
- 连续子数组和等于 `k`，尤其数组里可能有负数。
- 当前答案可以由左侧累计信息和右侧累计信息组合出来。

### 常用模式

- 前缀和把重复计算提前放到构造阶段。
- 二维前缀和可以理解为状态复用：当前格子复用上方、左方、左上方已经算好的结果。
- 二维矩阵前缀和常在外圈加一圈 `0` padding，统一公式，减少边界判断。
- 前缀积和前缀和是同一类思想。
- “除了自己以外的所有元素”可以拆成左边贡献和右边贡献。
- 前缀和 HashMap 常见初始化是 `{0: 1}`，表示空前缀，用来处理从下标 0 开始的子数组。

## 差分数组

### 详细笔记

- 详细整理见 [差分数组基础](difference_array_basics.md)。

### 识别信号

- 多次对连续区间增加或减少。
- 状态变化只发生在区间左右边界。
- 最后统一按位置扫描并恢复累计状态。
- 坐标范围较小，或者可以只排序发生变化的事件点。

### 基本原则

- 半开区间 `[start, end)`：`changes[start] += value`，`changes[end] -= value`。
- 闭区间 `[left, right]`：`changes[left] += value`，`changes[right + 1] -= value`。
- 差分数组保存变化量，必须通过前缀累加才能得到实际状态。
- 如果只关心每个位置的聚合数量，不关心具体对象是谁，差分通常比维护活动对象的堆更直接。
- 坐标巨大时不要创建完整数组，应考虑事件排序或坐标压缩。

## 滑动窗口

### 详细笔记

- 详细整理见 [滑动窗口基础](sliding_window_basics.md)。

### 识别信号

- 连续子数组 / 连续子串。
- 最长、最短、满足某个条件。
- 窗口内和、窗口内频率、不重复、至多/至少 `k` 个。

### 基本原则

- 滑动窗口本质是维护 `nums[left:right + 1]` 或 `s[left:right + 1]`。
- `right` 负责扩张窗口，`left` 负责收缩窗口。
- 做题前先定义窗口状态，例如 `window_sum`、字符频率、是否重复、不同字符数量。
- 窗口长度是 `right - left + 1`，因为左右端点都包含。
- 移动 `left` 前，要先把 `nums[left]` 或 `s[left]` 从窗口状态中移除。
- 普通滑动窗口通常需要状态有单调性；如果子数组和里有负数，优先考虑前缀和 + HashMap。

### 固定长度窗口

- 题目直接给 `k` 时，通常是固定长度窗口。
- 超过长度后移除最左边元素。
- 判断最近 `k` 个元素里是否出现过某个值，可以维护大小最多为 `k` 的 `set`。
- 窗口收缩有时放在加入当前元素之后，因为加入后的窗口是给下一个下标使用的。

### 可变长度窗口

- 可变窗口先明确窗口什么时候“不合法”或什么时候“已经满足条件”。
- 最长型：先收缩到合法，再更新 `max(answer, right - left + 1)`。
- 最短型：满足条件时，边收缩边更新 `min(answer, right - left + 1)`。
- 无重复子串：用 `set` 维护窗口；如果 `s[right]` 已经在窗口里，就不断移除 `s[left]` 直到合法。
- 一个重复字符可能需要移除多个左侧字符，所以经常用 `while` 收缩，而不是一次 `if`。

### 频率与覆盖

- 最多替换 `k` 次变成同一个字符：`窗口长度 - 窗口内最高频字符次数 <= k`。
- 判断是否包含某个排列：固定长度窗口 + 字符频率抵消。
- 固定长度窗口中，`>= k` 和 `> k` 对应不同流程；不能只改符号不改检查位置。
- 最短覆盖子串：`freq[ch]` 表示还缺多少，`matched == target` 表示窗口已经覆盖目标。
- 覆盖类窗口中：`freq[ch] > 0` 表示还缺，`== 0` 表示刚好满足，`< 0` 表示窗口里多了。

### 单调队列与特殊窗口

- 固定窗口最大值可以用单调队列维护候选最大值下标。
- 队列对应的值从大到小，队首就是当前窗口最大值。
- 单调队列通常存下标，因为需要判断队首是否已经滑出窗口。
- 如果最终答案一定是长度为 `k` 的连续窗口，并且数组有序，可以考虑二分窗口左边界。
- 二分固定窗口起点时，比较当前窗口左端 `arr[mid]` 和右移后会进入窗口的 `arr[mid + k]`。
- 找最接近 `x` 的 `k` 个元素时，不要用窗口 `sum` 判断接近程度；总和接近不代表每个元素都更接近 `x`。

## 二分查找

### 详细笔记

- 详细整理见 [二分查找基础](binary_search_basics.md)。

### 识别信号

- 排序数组中要求 O(log n)。
- 找插入位置、找左右边界。
- 找最小速度、最小容量、最小天数。
- 答案范围很大，但可以判断某个答案是否可行。

### 普通二分

- 先明确搜索区间是闭区间还是左闭右开。
- 普通闭区间二分：`while left <= right`。
- 每次排除 `mid` 本身，所以移动边界时通常写 `left = mid + 1` 或 `right = mid - 1`。
- Guess Number 中，`guess(mid) == 1` 表示猜小了，答案在右边；`guess(mid) == -1` 表示猜大了，答案在左边。

### lower_bound

- Search Insert Position 本质是 `lower_bound`：找第一个 `>= target` 的位置。
- `lower_bound` 中如果 `nums[mid] >= target`，`mid` 可能就是答案，所以保留 `mid`，写 `right = mid`。
- 左闭右开写法中，`right = len(nums)` 可以表示插入到末尾。
- 在有序时间戳列表里找不超过查询时间的最近记录，本质是找第一个 `> timestamp` 的位置，再减一。
- 左闭右开区间 `[left, right)` 中，`right` 不属于搜索范围，所以缩右边界写 `right = mid`，不是 `right = mid - 1`。
- Time Based Key-Value Store 可以用 HashMap 按 `key` 分组，再在每个有序时间戳列表里二分。

### 旋转排序数组

- 旋转排序数组的结构二分不一定有明确的 `target`，有时是在找旋转点或最小值。
- 找最小值时，可以比较 `nums[mid]` 和 `nums[right]`。
- 如果 `nums[mid] > nums[right]`，说明最小值一定在 `mid` 右边，写 `left = mid + 1`。
- 否则 `mid` 可能就是最小值，不能排除，写 `right = mid`。
- 在旋转数组中找 `target` 时，每一轮先检查 `nums[mid] == target`，再判断哪一半有序。
- 如果左半边有序，用 `nums[left] <= target < nums[mid]` 判断 `target` 是否在左半边。
- 如果右半边有序，用 `nums[mid] < target <= nums[right]` 判断 `target` 是否在右半边。
- `left` 和 `right` 是闭区间边界，仍然可能是答案；`mid` 已经被单独检查过，所以后续范围判断不再包含 `mid`。
- 如果数组允许重复元素，遇到 `nums[left] == nums[mid] == nums[right]` 时无法判断哪一半有序，只能先 `left += 1`、`right -= 1`。
- 允许重复元素的旋转数组二分，最坏时间复杂度可能退化到 O(n)。
- 这类题代码量可能比普通二分少，因为它不是在处理 `nums[mid] == target`，而是在判断答案落在哪一侧。

### 二分答案

- 二分答案题可以理解成“二分模板 + 判定函数”。
- 循环本身通常固定，真正差异在如何设计 `check(mid)`。
- 设计 `check(mid)` 时先问：`mid` 代表什么，给定 `mid` 能否验证可行，`mid` 越大是越容易还是越难，要找第一个 True 还是最后一个 True。
- 如果答案越大越容易满足，通常是在找最小可行答案，也就是第一个 True。
- 找最小可行答案时，`check(mid)` 为 True 不能写 `right = mid - 1`，因为 `mid` 可能正是答案；应该保留 `mid`，写 `right = mid`。
- 找最小可行答案时，`check(mid)` 为 False 说明 `mid` 不可行，可以排除，所以写 `left = mid + 1`。
- 二分答案的左右边界要有实际含义：左边界是最小可能答案，右边界是一定可行的最大答案。
- 需要向上取整除法时，可以用 `(value + divisor - 1) // divisor`。

### 最大可行答案

- 整数平方根可以转化成“找最大的 `mid`，使得 `mid * mid <= x`”。
- 找最大可行答案时，True 时写 `left = mid`。
- 为了避免死循环，`mid` 要取右中位数：`(left + right + 1) // 2`。
- 核心模板：`while left < right`，True 时 `left = mid`，False 时 `right = mid - 1`。

### 矩阵二分

- 如果每行递增且下一行第一个数大于上一行最后一个数，可以把矩阵整体看成一维有序数组。
- 一维下标映射回二维坐标：`row = mid // cols`，`col = mid % cols`。
- 也可以先二分定位候选行，再在行内二分；关键是明确第一段二分结束后 `left` 指向什么。

### 经典边界

- 运货容量题中，最小容量是 `max(weights)`，因为必须装得下最重包裹。
- 运货容量题中，最大容量是 `sum(weights)`，因为一天装完一定可行。
- 二分答案里的 `check` 通常按题意模拟，不要改变题目要求的顺序。

## 链表

### 详细笔记

- 详细整理见 [链表基础](linked_list_basics.md)。

### 识别信号

- 题目给的是 `ListNode`，不能随机访问下标。
- 需要删除、插入、反转、交换节点。
- 可能改变 `head`。
- 需要找中点、找倒数第 n 个节点、判断是否有环。
- 需要合并两个或多个有序链表。
- 数组里的值可以作为下一个下标，并且要求不修改数组、O(1) 空间。

### 基本原则

- 链表题核心是指针重连，不是下标计算。
- 修改 `curr.next` 前，先保存 `nxt = curr.next`。
- 如果可能改变 `head`，优先考虑 dummy。
- 如果用了 dummy，最后通常返回 `dummy.next`。
- 反转链表最后返回 `prev`，不是原来的 `head`。

### 常用指针

- `prev / curr / nxt`：常用于反转、删除、重连。
- `slow / fast`：常用于找中点、判断环、删除倒数第 n 个节点。
- `dummy / tail`：常用于构造新链表或合并链表。
- `old_to_new`：常用于复制复杂链表，保存原节点到新节点的映射。

### 常见模板

- 普通遍历：`while curr`。
- 访问下一个节点：`while curr and curr.next`。
- 删除节点：dummy + `prev.next = curr.next`。
- 反转链表：先存 `nxt`，再改 `curr.next`，再移动指针。
- 局部反转链表：先保存区间前驱、区间头、区间后继；反转后 `prev` 是新区间头，原区间头变成新区间尾。
- 快慢指针：`fast` 每次两步，`slow` 每次一步。
- 数组模拟链表：`next_index = nums[index]`，重复数字可以看成环入口。
- 两数相加：dummy 构造结果链表，同时维护 `carry`。
- 复杂指针复制：第一遍创建节点并建立映射，第二遍补 `next` 和 `random`。

### 易错点

- 忘记保存 `next`，导致后半段链表丢失。
- 删除节点时错误移动 `prev`。
- 用了 dummy 但返回了 `head`。
- 反转链表后返回了原 `head`，而不是 `prev`。
- 局部反转结束后误用 `curr.next` 接回后半段；此时 `curr` 通常已经是 `None`。
- `while curr` 和 `while curr.next` 混淆，导致漏处理最后一个节点。
- 复制复杂链表时直接复用原节点，导致不是深拷贝。
- 链表数字如果是倒序存储，可以从头直接相加；如果是正序存储，通常需要栈或反转。
- Floyd 找环入口时，第二阶段相遇的位置本身就是答案，不一定要再取一次 `nums[position]`。

## 树

### 详细笔记

- 详细整理见 [树基础](tree_basics.md)。
- Trie 专项见 [Trie 前缀树基础](trie_basics.md)。

### 识别信号

- 题目给的是 `TreeNode`，每个节点有 `left` 和 `right`。
- 需要计算深度、高度、节点数、直径或判断平衡。
- 需要处理根到叶路径、路径和或最近公共祖先。
- 题目要求按层输出、找最浅节点或统计每一层。
- 题目明确说明是 Binary Search Tree。

### 基本原则

- 树是数据结构，DFS/BFS 是遍历策略，递归/`while` 是实现方式；不要把树题直接等同于递归题。
- 先根据题目要求的访问顺序选择 DFS 或 BFS，再决定使用递归、栈还是队列实现。
- 不要一次追踪整棵树，先定义 `dfs(node)` 的职责和返回值。
- 固定五问：`dfs(node)` 表示什么，空节点返回什么，左右子树返回什么，当前节点如何组合左右结果，最终从哪里启动递归。
- 递归通常分为：空节点的 base case、递归左右子树、组合左右结果。
- 父节点把状态传给孩子属于自顶向下；孩子把结果返回父节点属于自底向上。
- 判断当前节点是否满足根到节点路径上的条件时，把路径最大值、路径和等状态作为参数向下传递；状态属于当前路径，不能让不同分支共享一个全局值。
- 前序是根左右，中序是左根右，后序是左右根。
- 深度、高度、平衡、直径通常使用后序；BST 的有序性通常使用中序。
- 直径类题要区分两个状态：递归返回给父节点的是单边深度，全局答案可以使用左右两边之和。
- 递归既要返回正常结果又要传播失败状态时，可以使用正常范围之外的哨兵值；例如平衡树中用 `-1` 表示子树不平衡。
- 父节点的选择会限制孩子状态时，可以让每个节点同时返回多个条件结果；例如返回 `(rob, not_rob)`，由父节点按自己的选择读取对应状态。
- 树形 DP 通常使用后序遍历：先取得左右子树状态，再计算当前节点的各个状态；状态元组的顺序一旦定义就必须始终一致。
- 比较两棵树时同步递归两个对应节点；递归结果必须被返回或组合，否则深层的失败结果会被当前层丢弃。
- 子树匹配可以拆成“遍历所有候选起点 + 完整比较两棵树”；当前候选失败后仍要继续向下搜索。
- 按层处理使用 BFS，并在每层开始时保存 `level_size = len(queue)`。
- BFS 每层选第一个还是最后一个节点，取决于孩子的入队顺序；左孩子先入队时，每层最后处理的是最右侧节点。
- Python 的 BFS 队列优先使用 `deque`：`append()` 和 `popleft()` 都是 O(1)，而 `list.pop(0)` 是 O(n)。

### 路径与 BST

- 路径回溯中，进入节点时 `append`，离开节点时 `pop`。
- 保存当前路径要复制列表，例如 `result.append(path[:])`。
- 叶子节点必须同时满足 `not node.left and not node.right`。
- BST 的中序遍历严格递增。
- 验证 BST 不能只比较父子节点，需要让每个节点满足祖先提供的严格范围 `lower < node.val < upper`；进入左子树更新上界，进入右子树更新下界。
- BST 中寻找两个节点的最近公共祖先：两者都小就向左，两者都大就向右，否则当前节点就是分叉点。
- 递归修改树时，可以让函数返回“处理后的子树根节点”，再由父节点用 `root.left/right = 递归返回值` 逐层接回；`node = None` 只会重绑局部变量，不能断开父节点中的连接。
- 当前节点是否删除取决于处理后的孩子时，使用后序遍历；删除返回 `None`，保留必须返回当前节点，最外层也要接住可能变化的根节点。
- BST 删除分三种情况：缺左孩子就返回右子树，缺右孩子就返回左子树，两个孩子都有时用后继节点或重连子树保持有序。
- 递归构造区域树时，用“区域位置 + 区域大小”定义子问题；区域可直接表示时返回叶子，否则拆分区域、递归构造孩子并接回当前节点。
- Trie 节点使用 `children` 保存“字符 -> 子节点”，使用 `is_end` 区分完整单词与普通前缀；`search` 要检查结束标记，`startsWith` 只检查路径。
- Trie 搜索遇到通配符时，普通字符只走对应孩子，通配符枚举全部孩子；任意分支成功即可返回，必须等所有分支失败后才能返回 `False`。

### 易错点

- 没有先说明递归返回值，导致左右结果和最终答案混在一起。
- base case 返回值与函数职责不一致。
- 混淆节点深度、节点高度，以及按节点还是按边计数。
- 路径题忘记回溯，导致一条分支污染另一条分支。
- 使用全局答案但没有在每次主函数调用前初始化。
- 默认树是平衡的；树最坏可能退化成链表，递归空间为 O(n)。

## Heap / Priority Queue

### 详细笔记

- 详细整理见 [Heap / Priority Queue 基础](heap_priority_queue_basics.md)。

### 识别信号

- 反复取出当前最小值或最大值。
- 第 `k` 大、第 `k` 小、Top K。
- 数据不断加入，并持续查询极值。
- 合并多个有序序列。
- 每轮选择最早结束、最早空闲或优先级最高的任务。
- 调度题同时存在“当前可执行”和“正在冷却”两类状态时，可以分别使用优先队列与按恢复时间排列的队列。
- 所有事件的未来到达时间预先已知时，通常先按到达时间排序并用指针推进；优先队列只维护当前已经到达的动态候选。

### 基本原则

- Python 3.12 的 `heapq` 基础接口是最小堆；最大堆通常把数值取负。
- 反复取当前最大值时，可以把原值取负后放入最小堆；入堆取负，出堆后再次取负还原。
- 堆只保证根是极值，不保证内部列表完全有序。
- 保留最大的 `k` 个元素，维护大小为 `k` 的最小堆；堆顶是当前入选集合中的淘汰门槛。
- 数据流中持续查询第 `k` 大时，最小堆只保留目前最大的 `k` 个元素，`heap[0]` 就是答案。
- 固定数组中查询第 `k` 大也可以使用相同的不变量；与数据流版本相比，只需在遍历结束后读取一次堆顶。
- 保留最小的 `k` 个元素，维护大小为 `k` 的最大堆。
- 距离最近的 `k` 个属于“保留最小的 k 个”；最大堆把当前最远候选放在堆顶，超过大小时优先淘汰它。
- 写元组入堆前，先说明每个字段的含义、升降序和相同优先级的处理规则。
- K 路归并时，每个来源通常只在堆中保留一个当前候选。
- 多个有序序列只需取前若干个全局最优元素时，弹出一个来源的候选后，只补入该来源的下一个候选，并在答案数量足够后停止。
- Task Scheduler 一类题中，cooldown 负责可执行资格，最大堆负责从可执行任务中选择剩余次数最多者；不要混成同一个排序规则。
- 如果当前元素下一轮暂时不能再次使用，可以先把它留在堆外；限制只持续一轮时，一个 `previous` 状态通常足够。
- 堆顶候选因当前局部限制暂时不可用时，可以改用第二候选，并把未使用的堆顶原样放回；若没有第二候选，则当前状态无法继续扩展。
- 时间模拟中如果当前候选为空，不要逐单位空转，应直接跳到下一个事件发生的时间。

### 常用操作

- 原地建堆：`heapq.heapify(values)`，O(n)。
- 插入：`heapq.heappush(heap, value)`，O(log n)。
- 弹出最小值：`heapq.heappop(heap)`，O(log n)。
- 查看最小值：`heap[0]`，O(1)。
- 固定大小堆：先 `heappush`，再在 `len(heap) > k` 时 `heappop`。

### 易错点

- 把堆的内部列表当成排序数组。
- 最大堆入堆或出堆时忘记处理负号。
- 把 `heapq.heapify()` 的返回值赋给堆；它原地修改列表并返回 `None`。
- Top K 选错最小堆或最大堆。
- 固定大小堆误写成 `len(heap) >= k`，导致只保留 `k - 1` 个元素。
- 元组优先级相同时，Python 继续比较了不可比较的对象。

## Backtracking

### 详细笔记

- 详细整理见 [Backtracking / 回溯基础](backtracking_basics.md)。

### 识别信号

- 返回所有子集、组合、排列或字符串分割方案。
- 需要一步一步构造答案，每一步都有多个候选。
- 当前选择会限制后续选择，无效分支可以提前停止。
- 题目要求在棋盘中寻找路径、放置元素或构造所有合法状态。
- 输入规模较小，允许枚举指数级搜索空间。

### 四个固定问题

1. `path` 表示什么？
2. 当前有哪些候选选择？
3. 什么选择不合法？
4. 什么时候收集答案？

### 基本原则

- 回溯通常按照“做选择、递归、撤销选择”的顺序。
- 保存可变路径时使用 `path[:]`，不能直接保存同一个列表引用。
- 顺序不重要的子集和组合通常使用 `start`，防止生成顺序不同的重复结果。
- 顺序重要的排列通常使用 `used`，每层检查所有尚未使用的元素。
- 下一层传 `i` 表示当前元素可以复用，传 `i + 1` 表示当前下标只能使用一次。
- 数值目标题常用 `remaining` 表示距离目标还差多少。
- 棋盘题进入位置时标记，离开当前路径时必须恢复。

### 去重与剪枝

- 输入含重复元素时通常先排序。
- 组合型同层去重常用 `if i > start and nums[i] == nums[i - 1]: continue`。
- 排列去重需要结合排序和 `used` 状态。
- 排序后当前值已经超过剩余目标，可以 `break`。
- 剩余候选数量不足、括号前缀不合法、棋盘发生冲突时都可以提前剪枝。
- 剪枝必须能证明整棵分支都不可能产生答案。

### 易错点

- 忘记 `path.pop()` 或恢复 `used`、棋盘、集合状态。
- `result.append(path)` 没有复制路径。
- 混淆组合的 `start` 与排列的 `used`。
- 当前元素是否复用时把 `i` 和 `i + 1` 传反。
- 子集、组合、排列的答案收集位置混用。
- 去重条件误写成 `i > 0`，错误跳过深层合法选择。
- 没有排序就尝试通过相邻元素完成去重。

## Graph

### 详细笔记

- 详细整理见 [Graph / 图基础](graph_basics.md)。

### 识别信号

- 节点之间存在道路、连接、关注、依赖或转换关系。
- 需要判断两个状态是否可达。
- 需要统计连通分量、岛屿、区域数量。
- 需要寻找无权最短步数或传播所需时间。
- 任务之间存在先后依赖，需要判断环或返回执行顺序。
- 边不断加入，需要维护哪些节点属于同一集合。

### 建图前固定问题

1. 节点是什么？
2. 边是什么？
3. 有向还是无向？
4. 有权还是无权？
5. 输入是边列表、邻接表、矩阵还是网格？
6. 图是否可能不连通？
7. `visited` 表示什么？

### 基本原则

- 无向图的边需要在邻接表中添加两个方向；有向图只添加题目规定的方向。
- 邻接表适合稀疏图，遍历复杂度通常为 O(V + E)。
- 图可能有环，基础 DFS/BFS 必须维护 `visited`。
- DFS 决定进入节点时标记；BFS 决定入队时标记。
- 图可能不连通，处理全部节点时需要从每个未访问节点重新启动遍历。
- 网格可以看作隐式图，格子是节点，上下左右移动是边。

### DFS 与 BFS

- 可达性、连通分量：DFS 或 BFS 都可以。
- 无权最短路径、最少步数、按层传播：优先 BFS。
- 多个起点同时扩散：所有起点一起加入队列，使用多源 BFS。
- 图很深时，递归 DFS 可能超过 Python 递归限制，可以改用显式栈。

### 常用进阶模式

- 无向图判环：DFS 记录父节点，遇到已访问且不是父节点的邻居说明有环。
- 有向图判环：区分未访问、当前路径中、已经完成三种状态。
- 课程依赖：统计入度，使用 Kahn BFS 完成拓扑排序。
- 动态连通：使用 Union-Find 的 `find` 和 `union`。
- 无权最短路：BFS。
- 非负权最短路：Dijkstra + 最小堆。

### Graph 与 Backtracking 的 visited

- 普通 Graph 可达性中，节点一旦处理完成通常永久保留在 `visited`，不撤销。
- Backtracking 枚举路径时，标记可能只属于当前路径，递归返回后需要撤销。
- 是否撤销取决于状态含义，而不是代码是否使用 DFS。

### 易错点

- 无向边只添加一个方向，或把有向边错误添加成双向。
- 忘记 `visited`，在有环图中无限循环。
- BFS 等到出队才标记，导致同一节点重复入队。
- 只从节点 0 开始，漏掉其他连通分量。
- 网格题混淆 `row`、`col` 或漏写边界。
- Course Schedule 把 prerequisite 和 course 的边方向建反。
