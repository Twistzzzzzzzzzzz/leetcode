# 344. Reverse String

## 题目

给定一个字符数组 `s`，要求原地反转整个数组。

LeetCode 要求不要返回新数组，而是直接修改传入的 `s`。

## 思路

这题适合使用左右双指针。

定义：

- `left` 指向当前最左边还没有处理的位置
- `right` 指向当前最右边还没有处理的位置

每次交换 `s[left]` 和 `s[right]`，然后让：

```python
left += 1
right -= 1
```

也就是从两端向中间收缩。

当 `left >= right` 时，说明已经交换完成。

## 如何识别左右双指针

如果题目出现这些特征，可以先想到左右双指针：

1. 需要原地处理数组或字符串。
2. 操作和两端元素有关。
3. 需要从左边和右边同时向中间推进。

比如：

- 反转数组 / 字符串
- 判断回文
- 两端夹逼查找
- 首尾元素成对交换

这题的第一反应可以是：

```text
反转 = 最左边和最右边交换，然后继续处理里面一层
```

所以自然可以想到 `left` 和 `right` 两个指针。

## 代码

```python
class Solution:
    def reverseString(self, s: list[str]) -> None:
        left = 0
        right = len(s) - 1

        while left < right:
            s[left], s[right] = s[right], s[left]
            left += 1
            right -= 1
```

## 为什么是 `while left < right`

双指针从两边向中间走时，循环条件通常写：

```python
while left < right:
```

不能简单写成 `left != right`。

因为数组长度是偶数时，左右指针会直接错过：

```text
left:  1 -> 2
right: 2 -> 1
```

它们不会停在同一个位置。

如果只判断 `left != right`，偶数长度时可能会继续执行多余的交换，甚至把已经反转好的内容又换回去。

所以左右指针向中间收缩时，更稳的结束条件是：

```text
只要 left 还在 right 左边，就继续处理
```

也就是 `left < right`。

## 复杂度

- 时间复杂度：O(n)
- 空间复杂度：O(1)

每个字符最多参与一次交换，并且只使用了两个指针。

## 心得

1. 遇到“反转”“回文”“首尾交换”这类题，可以先想到左右双指针。
2. 左右双指针的核心是：从两端开始处理，然后向中间收缩。
3. 循环结束条件一般写 `while left < right`，因为偶数长度时左右指针会错过，不一定会相等。
