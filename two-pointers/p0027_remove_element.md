# 27. Remove Element

## 题目

给定一个数组 `nums` 和一个值 `val`。

要求原地移除数组中所有等于 `val` 的元素，并返回剩下元素的数量。

注意：题目只关心返回的长度 `k`，以及 `nums` 前 `k` 个位置里的元素。后面的内容不重要。

## 思路

这道题可以理解为：把所有“不等于 `val` 的元素”往数组前面搬。

用一个变量 `write_index` 表示“下一个有效元素应该放在哪里”。

遍历数组时：

1. 如果当前元素等于 `val`，跳过它。
2. 如果当前元素不等于 `val`，就把它写到 `nums[write_index]`。
3. 写完后，`write_index += 1`。

最后 `write_index` 就是剩余元素的数量。

这里不要把它想成“从数组里删除元素”。

更准确的理解是：重新构造数组前面的有效区间。

也就是把所有需要保留的数字按顺序移动到最前面，后面的内容题目不关心。

## 代码

```python
class Solution:
    def removeElement(self, nums: list[int], val: int) -> int:
        write_index = 0

        for num in nums:
            if num != val:
                nums[write_index] = num
                write_index += 1

        return write_index
```

## 例子

```text
nums = [3, 2, 2, 3]
val = 3
```

遍历时只保留不等于 `3` 的元素：

```text
2, 2
```

所以返回：

```text
2
```

并且数组前 2 个位置是：

```text
[2, 2]
```

## 复杂度

- 时间复杂度：O(n)
- 空间复杂度：O(1)

因为只遍历一次数组，并且是在原数组上修改，没有创建新的数组。

## 心得

1. 这题确实比较直接，核心是“原地覆盖”。
2. `write_index` 可以理解为有效区间的下一个写入位置。
3. 遇到“原地删除 / 原地保留某些元素”的题，可以先想：用一个指针记录下一个应该写入的位置。
4. 这题不是把元素真正删除，而是把所有要保留的数字移动到数组最前面，形成一个新的有效前缀。
