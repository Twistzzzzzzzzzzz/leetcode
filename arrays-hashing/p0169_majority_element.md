# 169. Majority Element

## 题目

给定一个数组 `nums`，返回其中的多数元素。

多数元素指的是：出现次数大于 `n / 2` 的元素。

题目保证多数元素一定存在。

## 思路

这道题可以用 Boyer-Moore 投票法。

关键条件是“绝对多数”：

```text
多数元素出现次数 > 其他所有元素出现次数之和
```

所以每次拿一个多数元素和一个非多数元素互相抵消，最后剩下的仍然一定是多数元素。

投票法的写法：

1. `candidate` 表示当前候选人。
2. `vote` 表示当前候选人的票数。
3. 遇到相同元素，`vote + 1`。
4. 遇到不同元素，`vote - 1`。
5. 如果 `vote == 0`，说明当前候选人已经被抵消完，可以换新的候选人。

## 代码

```python
class Solution:
    def majorityElement(self, nums: list[int]) -> int:
        candidate = nums[0]
        vote = 0

        for num in nums:
            if vote == 0:
                candidate = num

            if num == candidate:
                vote += 1
            else:
                vote -= 1

        return candidate
```

## 例子

```text
nums = [2, 2, 1, 1, 1, 2, 2]
```

可以把不同元素互相抵消：

```text
2 和 1 抵消
2 和 1 抵消
1 和 2 抵消
```

因为 `2` 是绝对多数，最后一定还会剩下 `2`。

## 复杂度

- 时间复杂度：O(n)
- 空间复杂度：O(1)

只遍历一次数组，并且只使用 `candidate` 和 `vote` 两个变量。

## 心得

1. 当题目出现“绝对多数”时，每个候选人具体有多少票已经没那么重要，因为总有一个元素的票数超过所有剩余元素之和。
2. 因此可以想到投票和抵消思想：遇到相同元素 `vote + 1`，遇到不同元素 `vote - 1`。
3. 当 `vote` 归零，说明当前 `candidate` 被抵消完，可以换新的 `candidate`。
4. 这道题能用 O(1) 空间，是因为题目保证多数元素一定存在。如果不保证存在，最后还需要再遍历一次验证候选人是否真的超过一半。
