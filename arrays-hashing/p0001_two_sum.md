# 1. Two Sum

## 题目

给定一个整数数组 `nums` 和一个目标值 `target`。

在数组中找出两个数，使它们相加等于 `target`，并返回这两个数的下标。

题目保证只有一个答案，并且同一个元素不能使用两次。

## 思路

最直接的做法是两层循环：

1. 固定第一个数字。
2. 再去后面找有没有另一个数字能和它加起来等于 `target`。

这种做法需要重复比较很多次，时间复杂度是 O(n²)。

想优化，就要换一种数据结构。

这道题可以用字典 `dict` 来记录“我正在等待哪个数字出现”。

例如当前数字是 `num`，目标值是 `target`，那么另一个需要的数字就是：

```text
target - num
```

遍历数组时：

1. 如果当前数字已经在字典里，说明之前有一个数字正在等它，直接返回两个下标。
2. 如果当前数字不在字典里，就把 `target - 当前数字` 存进字典，表示后面要找这个数字。

## 代码

```python
class Solution:
    def twoSum(self, nums: list[int], target: int) -> list[int]:
        needed = {}

        for index, num in enumerate(nums):
            if num in needed:
                return [needed[num], index]
            needed[target - num] = index

        return []
```

## 例子

```text
nums = [2, 7, 11, 15]
target = 9
```

第一次看到 `2`：

```text
9 - 2 = 7
```

说明后面需要找 `7`，所以记录：

```python
needed[7] = 0
```

第二次看到 `7`，发现 `7` 已经在 `needed` 里，说明答案找到了：

```python
[needed[7], 1]
```

也就是：

```python
[0, 1]
```

## 复杂度

- 时间复杂度：O(n)
- 空间复杂度：O(n)

只遍历一次数组，所以时间复杂度从暴力解法的 O(n²) 优化到 O(n)。

## 心得

1. 如果想优化算法，可能需要换一种数据结构。
2. 优化其实就是找一个只遍历一次的解法，那就得找表达式之间的关系。这里的关系是：`另一个数 = target - 当前数字`。
3. 这道题的优化本质是把“在数组里线性查找另一个数”，变成“在哈希表里查找另一个数”。也就是单次查找从 O(n) 变成 O(1)。
4. 优化 = 找出暴力解中重复做的事情，然后用合适的数据结构或状态把它保存下来。
5. 如果题目在问“数组中是否存在两个数或多个数满足某种关系”，可以想到用哈希表保存已经见过的信息，或者保存后面需要匹配的关系。
6. 写哈希表时必须先明确字典含义：是存 `value -> index`，还是 `complement -> index`。
